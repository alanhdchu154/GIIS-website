# Lesson video skill

> Take a folder containing `script.json` + `slides/` and produce a finished MP4 with English Aria narration and burned-in subtitles. Use this for every GIIS teaching video.
>
> V2 production quality rules live in `tools/lesson-video/QUALITY_FLOW.md`.
> Read that file before creating, reviewing, rendering, or uploading any
> lesson. Claude Code can draft lessons, but the release gate decides whether
> they are uploadable.

## When to use this skill

Trigger on any of:
- "merge the lesson", "build the video", "render the MP4"
- "stitch audio + slides", "make the lesson video"
- "rebuild the video with the new audio"
- After Claude finishes writing a new `script.json` and generating slides — invoke this so the user gets a finished MP4 without doing extra work.

## Two entry points (pick the right one)

### `make_lesson.py` — full pipeline (Mac-side)

**Use when:** the lesson is new, or the script changed and the audio needs regenerating.
**Where to run:** the user's Mac (the sandbox can't reach Microsoft's TTS endpoint).

```bash
cd /Users/alanhdchu/giis-website
pip install edge-tts imageio-ffmpeg          # one-time
python3 tools/lesson-video/make_lesson.py teaching-videos/<lesson-folder>/
```

What it does:
1. For every section without a corresponding `audio/<id>.mp3`, calls edge-tts with the voice + rate from `script.json` and saves the MP3.
2. Hands off to `merge_lesson.py` to do everything below.

Idempotent — re-running only re-synthesizes MP3s that don't exist. Delete an MP3 to regenerate it.

### `merge_lesson.py` — merge only (sandbox or Mac)

**Use when:** audio MP3s already exist (they were generated by a previous `make_lesson.py` run, or you just tweaked slides without changing the script). Claude can run this directly from sandbox — no round-trip needed.

```bash
python3 tools/lesson-video/merge_lesson.py teaching-videos/<lesson-folder>/
```

This is the script Claude invokes when the user says "merge the lesson" in chat.

## Inputs the skill expects

```
<lesson-folder>/
├── script.json              # required — see schema below
├── slides/
│   ├── 01_<id>.png          # required — one PNG per section
│   └── ...
├── contact-sheet.jpg        # required before upload — visual QA thumbnail sheet
├── _review_A*.json          # required before upload — PhD/peer content check
├── _review_B*.json          # required before upload — adversarial student check
├── _review_C*.json          # required before upload — citation/source/license check
├── assets_manifest.json     # required if sourced/generated images are used
├── learning_check.json      # recommended — quiz/checks tied to lesson misconceptions
├── audio/                   # made by make_lesson.py / synth_audio_local.py
│   ├── 01_<id>.mp3
│   └── ...
├── intro_music.wav          # optional — copy from this skill folder
└── outro_music.wav          # optional — copy from this skill folder
```

The slide filename's prefix MUST match the section id (e.g. `id: "08_pause1"` → `slides/08_pause1.png`).

## V2 quality bar

Use the following tools/skills as quality layers:

- `presentations:Presentations` standard for slide story: one claim/action per slide, coherent contact sheet, no filler layouts.
- `imagegen` only for hook illustrations, thumbnails, and low-precision conceptual scenes. Precision diagrams should be deterministic code in `build_slides.py`.
- `browser:browser` for Learn Portal / HTML / embed QA when a local page is involved; otherwise generate and inspect `contact-sheet.jpg`.
- `engineering:testing-strategy` for `learning_check.json`, A/B tests, and release rubric.

Upload gate defaults:

- score >= 90
- `pass` or `pass_with_minor_notes`
- no major/critical audit issues
- MP4 + transcript + contact sheet present
- PhD/peer + adversarial student + citation/source reviewer artifacts present
- AP lessons require citation/source review

Run before upload:

```bash
python3 tools/lesson-video/lesson_release_gate.py --pending
```

### `script.json` schema

```json
{
  "course": "string",
  "module": "string",
  "voice": "en-US-AndrewNeural",
  "voice_rate": "-3%",
  "intro_music_seconds": 3,
  "outro_music_seconds": 3,
  "sections": [
    { "id": "01_title", "text": "..." },
    { "id": "02_hook",  "text": "..." }
  ]
}
```

### Voice defaults per subject (use these unless the user says otherwise)

When Claude writes a new `script.json`, pick `voice` from this table — each
subject gets a distinct neural voice so the curriculum feels like a real
school (different teacher per class) rather than one robot reading everything.

| Subject area                                          | Voice                       | Rationale |
|-------------------------------------------------------|-----------------------------|-----------|
| Math (Algebra, Geometry, Calc, Stats, Trig, Pre-Calc) | `en-US-AriaNeural`          | Warm + patient + clear. Math triggers anxiety; needs "we'll solve this together" tone. |
| Science (Bio, Chem, Physics, Env Sci, AP Bio)         | `en-US-EmmaNeural`          | Curious + energetic. "You won't believe this" wonder. |
| English / Literature (Eng I-IV, Composition, Writing) | `en-US-AndrewNeural`        | Best storytelling voice in the catalog. Suits literary analysis. |
| Social Studies (History, Govt, Econ, Geography)       | `en-US-ChristopherNeural`   | Documentary-style authority. Fits civics + history. |
| Psychology (Foundations, AP Psych, Cognitive, etc.)   | `en-US-BrianNeural`         | Thoughtful + warm. Psych needs "I'm listening carefully" tone. |
| PE / Health (Athletic, Health & Wellness, Sports Mgmt)| `en-US-JennyNeural`         | Coach energy. "Let's move." |
| Computer Science (AP CS A, Programming, Software Eng.)| `en-US-GuyNeural`           | Academic authority. Same voice as walkthrough Pathway scene — continuity. Distinct from Math's Aria. |
| Electives / Business (default)                         | `en-US-AriaNeural`          | Safe default when none of the above fits. |

**Rate**: stay at `-3%` for instructional clarity unless the user requests faster/slower.

**Bilingual** future expansion: when generating `zh-CN` versions, use the
*Multilingual* voices (e.g. `en-US-AndrewMultilingualNeural`) so the same voice
delivers both languages — preserves the "same teacher" feel across languages.

`intro_music_seconds` / `outro_music_seconds` are optional. If `intro_music.wav` exists and the value is > 0, the music plays under the title slide before narration. Same for outro.

## Pipeline that merge_lesson.py runs

1. Resolves an ffmpeg binary — system `ffmpeg` if present, otherwise `imageio_ffmpeg.get_ffmpeg_exe()` (a static binary shipped in the PyPI wheel).
2. Converts `audio/<id>.mp3` → mono 22050 Hz WAV. Re-converts if a stale 0-byte WAV is found.
3. Builds `subtitles.srt` by chunking each section's text proportionally to its audio duration (≤ 10 words per cue).
4. Builds master audio: `[intro_music] → section_1 → 0.4s gap → … → section_N → [outro_music]`.
5. Builds slide concat list — each slide is held for its section's audio + gap. Title slide is held during intro music; recap during outro.
6. Renders MP4 (1920×1080, H.264 CRF 22, AAC 128k, 30 fps) with subtitles burned in via libass.

Output: `<folder-name-with-underscores>.mp4` next to the script. Successive runs overwrite the same file.

## Audit and release gate

After slides/audio/MP4 exist, run:

```bash
python3 tools/lesson-video/audit_lessons.py teaching-videos/<lesson-folder>/
python3 tools/lesson-video/lesson_release_gate.py teaching-videos/<lesson-folder>/
```

If the release gate returns `needs_revision`, revise the script/slides/reviewer
artifacts before YouTube upload. Do not treat "MP4 exists" as sufficient.

## Required dependencies

- Python 3.10+
- `edge-tts` (PyPI) — only needed by `make_lesson.py` for synthesis
- `imageio-ffmpeg` (PyPI) — bundles a static ffmpeg binary; means **no `brew install ffmpeg` required**
- system `ffmpeg` is used if available, but optional

## Failure modes / pitfalls

- `audio/<id>.wav` exists but is 0 bytes — automatically deleted and re-converted from MP3.
- A section in `script.json` with no matching slide — fails fast with a clear message.
- TTS network failure (sandbox blocks Bing) — `make_lesson.py` will throw a DNS error on first synth call. Run on a machine with internet.
- Mounted filesystems sometimes deny `unlink` of the temporary `_silence.wav` — handled silently.
