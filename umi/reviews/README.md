# Umi Reviews

Use this folder for Umi's review of Claude Code output, human patches, or production behavior.

The review should lead with risks and fixes, not praise. Alan needs clear signal, not a ceremony.

## Template

```md
# Review: <task or diff>

## Verdict

- Ship / revise / hold

## Findings

- Severity: blocker / major / minor
- File or behavior:
- Why it matters:
- Suggested fix:

## GIIS Lens

- Trust:
- Transparency:
- Results:
- Operations:

## Verification

- commands run
- browser checks
- tests not run and why

## Roadmap Status

- updated / needs update
```

## Review Priorities

1. Parent trust and truthful school claims
2. Student records, transcript, diploma, and payment safety
3. Parent transparency and operational clarity
4. Build/test correctness
5. Maintainability and scope control
